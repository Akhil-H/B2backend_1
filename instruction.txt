# Project Working Instructions & History

## Recent Implementation: AWS S3 Image Upload
Date: 2026-02-21

### 1. Backend Changes (server.js)
- Installed `@aws-sdk/client-s3`, `multer`, and `multer-s3`.
- Configured `S3Client` for AWS connection.
- Implemented `multerS3` storage for direct uploads to S3.
- Updated `POST /api/salons` to handle `multipart/form-data` and save the S3 `location` URL to MongoDB.

### 2. Frontend Changes (public/)
- **index.html**: Replaced the URL text input with a file input (`<input type="file">`) for device image selection.
- **script.js**: Updated form submission to use `FormData` object instead of JSON to support file uploads.

### 3. Environment Setup (.env)
- Added AWS configuration keys:
  - `AWS_ACCESS_KEY_ID`
  - `AWS_SECRET_ACCESS_KEY`
  - `AWS_REGION`
  - `AWS_S3_BUCKET_NAME`

---

## 🛠 Required Configuration
1. Fill in actual AWS credentials in the `.env` file.
2. Ensure S3 Bucket has "Public access" enabled or appropriate Bucket Policy if using `public-read` ACL.
3. Ensure CORS is configured on the S3 bucket to allow the origin.
